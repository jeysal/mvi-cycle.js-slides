* Model-View-Intent establishes a unidirectional data flow * Intent maps stream of user clicks, key presses etc. to stream of "show customers", "create new customer" * Translates from user's mental model to application model * Model maps stream of user intent to stream application state transformations * View maps stream of application state to stream of visual representations * User maps stream of screen content to incremental stream of mental model * We have to implement the intent, model and view * Those are just functions, could be consolidated into one or split up further
* How does MVI translate to code in Cycle.js? * Example code from Cycle.js website * `main` function gets DOM events / http responses / w/e input from sources * Returns sinks (DOM elements / http requests) * Application code maps sources to sinks * Drivers map sinks to sources * Explain intent (`add$`), model (`count$`) and view (return value) * Show live changes of numbers
